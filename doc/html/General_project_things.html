cvs repository

Setup and Starting

To check out a new project:

    $ cvs -d cmertz@unix.andrew.cmu.edu:/afs/andrew.cmu.edu/scs/ri/Project-2011/pothole/cvsroot checkout pothole_app

You need to replace "cmertz" with your own andrew username.

For your convenience, create the following in .bashrc:

    export CVSROOT=cmertz@unix.andrew.cmu.edu:/afs/andrew.cmu.edu/scs/ri/Project-2011/pothole/cvsroot

of course, change "cmertz" to your username.
The check out command simplifies to:

    $ cvs checkout pothole_app

You should also declare you favorite editor in .bashrc:
     export CVSEDITOR='emacs'

Using cvs


If you modify a file and you want to commit it so that everyone else can access it:
   
    $  cvs commit

If you want to update your files to get the changes others made:

    $ cvs up

To add a file or directory

    $ cvs add file_or_dir_name


Documenting


Make continous notes about what you are doing.

Once you have finished a task (e.g. investigating mapping applications), write one summary documentation in html and link that file to index.html.
Make sure that a table of content is at the top of the documentation and that table of content is copied to index.html.

Suggested html editor: Seamonkey
