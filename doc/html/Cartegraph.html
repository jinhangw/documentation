Work with street centerlines in ArcMap and Cartegraph


The scored data points has to be associated to the street centerline data. This process involves two database system. On one side, data is stored in geodatabase, and visualized on ArcMap; on the other side, data is stored in Oracle database and displayed in Cartegraph. So, our scored data points are first imported into the geodatabase, and snapped to the street centerline map. Then, the scores are updated in the Oracle database associated with Cartegraph.

This documentation discuss the approach in three parts:
1. prepare the ArcMap/geodatabase
2. prepare the Cartegraph

3. connection between ArcMap and Cartegraph

For the Match Option, select WITHIN_A_DISTANCE. Set Search Radius as 60, unit as Feet. Click OK. Wait for it to process.

The Inspections and Events recordsets would be considered child recordsets. For each segment record in the main recordset there can be many inspection (from different years)or event records.


How to generate OCI from imported score

We currently scale our score to 100, and use it as extent.

Build the spreadsheet compatible with the database structure.

1. structure

The ‘Inspection’ recordset is considered child recordset of the ‘Segment’. The ‘Inspection’ recordset is the parent recordset to the ‘Distress Sample’, and the grand parent to the ‘Distress’ recordset.

Accordingly, for each ‘Distress’ record in the spreadsheet, it should contain 3 ID fields to identify the parent, grandparent, grand-grandparent. (i.e. Distress samples, Inspection, and Segment.)

2. fields

The spreadsheet should combine all necessary information in the recordset hierarchy. This spreadsheet will be the same spreadsheet which gets imported to recordsets of different levels. The system differentiates them by fillering out different fields.

3. format

Date: MM/DD/YYYY as in Cartegraph

Note: In Excel, =DATE(LEFT($D4,4), MID($D4,5,2), MID($D4,7,2))

Extent: In Cartegraph, Extent is a number between 0 to 100; while our score between 0 to 1. Thus we adjust our averaged score by multiplying 100 to represent the Extent.

Severity: a lookup field which takes a value of "High”, “Moderate”, or “Low". No other value is accepted.

ID: A recommended way to create ID for Distress Samples is date+rank. For example, the first distress sample on Oct. 4th, 2012 is 100420121.

Type: Put Random for all records.

*All fields should contain only values, rather than fomula.

===============Arcmap to Cartegraph==============

Connection and interaction between ArcMap and Cartegraph



1. Share Street Centerline shapefile with Cartegraph

1.1 Open ArcCatalog.

In ArcCatalog, browse to the shapefile you want to connect to Cartegraph. Click on the Preview tab on the right. Choose to preview Table, instead of Geography on the bottom. In the left penal, right click on the shapefile name > properties… > Fields tab. Click on each record to see the type and other properties. This is to help prepare data normalization in Cartegraph.


1.2 Create a new Categraph database

Open MS SQL Server Management Studio, connect to your instance. (Delete the trashed database). Right click on Database > New Database… > Type the Database Name as Cartegraph.

Go to Cartegraph, Administrator page. Action > Format Database Wizard or . Click Next. Select the Project Home to the location of the shared Cartegraph database. In Select a default License Server, check the box for Product-based Licensing, then check Don’t use a default server… Click Next. Select the module PavementView. Click Next, Next, Finish.

Open the Tools > Shared Options dialog. Go to GIS Integration, Check the box to Enable GIS Integraion.


1.3 Connect to a geodatabase

In Cartegraph Navigator, Administrator page, right click on GeoDB connections, select New. In Select Geodatabase Object, browse to the shapefile (StreetCenterlineBackup, in our case). In Cartegraph Recordset: <select a recordset>, choose segment.

Find our FIDID field, click on the Cartegraph Fields cell on FIDID record, select ID, click Ok. Wait for it to process. Click on Segment page in the Navigator, we can see a form shows up (all empty fields except ID). And the record count changed at the bottom.


1.4 Add the field which is not included in any Cartegraph library

Go to Administrator page, unfold Recordsets, click on Segment, click on Add Fields button from the tool bar. Under General tab, Name the field. put the same name for ID. Choose the same Type as in the shapefile. Under Data Source tab, Put Column Name as the field name. Change Field Size as in the shapefile. Now go back to geoDB Connections, double click on the shapefile, find the field and choose the new created Cartegraph Field for it. This refresh process might take hours (2.5 hours for 23,000 records for updating one 50-digit text field; 2.5 mins for 3,000 records, in our case).


1.5 Add the field into the Form.

Go to Segment page. In the Form view, click on Edit to switch to the edit mood. Save the form as a new form. Make changes to the new Form, so that the default Form remains intact.




2. Import Score data/Image data into Cartegraph

2.1 Export Score data from ArcMap as csv file

In ArcMap, left penal, right click on the joined shapefile, > open attribute table > table options > export… click on Browse, rename the file, save as TXT file.

CSV file requirements:

The first column should be Cartegraph ID. This is to make the process as fast as possible. Keep only necessary columns. Especially do not mix two sets of data. i.e. Do not have image names column in the inspection (score) csv file.

In Excel, open the txt file. Check the box for delimited, and comma as the wizard guide you.

Insert a new column next to ‘tag’ (In this example, column D). type the fomula

=DATE(LEFT($D4,4), MID($D4,5,2), MID($D4,7,2))

to extract the first 8 digits and to convert into date.

2.2 Import into Cartegraph

In Cartegraph, Segment mode, Go to File > Import and Export > Format Builder. Primary Recordset as Segment\Inspection (Segment\Attachment for images). Select Field Separator and other parameters properly. Under Field Name, Put the first one as ..\ID. Put ALL columns as fields in Cartegraph. If any column is not needed, then add it as ‘filler’.


3. How to disconnect the shapefile

Backup the database before making any change. So, in Cartegraph….we can restore the data.

But the data is updated in ArcGIS. To be able to edit the shapefile, we have to disconnect the shapefile from Cartegraph GeoDB Connection.

Find the .xml file of the shapefile. Right click > Edit to open it in a notepad editor. Ctrl + F to find ‘CarteAssociation’. Delete the tag <CarteAssociation>(and the information inside the tag)</CarteAssociation>. Then in ArcMap, remove the old shapefile from Table of Contents, if it was there. Add the new shapefile. Now, you will be able to edit the shapefile.
