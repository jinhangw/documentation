Ground labeling software

Download Derek Hoiem's image pop software from this link: 

http://www.cs.illinois.edu/homes/dhoiem/projects/context/GeometricContext.zip 

Extract this package and put it into directory say, imagePop.

Also this package need an external package, which you need to download separately: 

people.cs.uchicago.edu/~pff/segment/

Extract it into imagePop directory too and compile it.

Open imagePop/GeometricContext/src/im2superpixels.m 

Change the following line: 

segcmd = 'segment 0.8 100 100'; 

into: 

segcmd = 'LD_LIBRARY_PATH=/usr/lib64 ../../segment/segment 0.8 100 100';

Open matlab 

cd imagePop/GeometricContext/src/; 

addpath(genpath('.')); 

photoPopupIjcv('../data/ijcvClassifier.mat', inputImageFileName, [], '../test_dir/')

It takes 15 secs per image of size 800x600.

Bad example and failure modes:

1.    Miss classifies partial cars, shadows as ground.

2.    Algorithm makes assumption about camera position, which is at human height and facing forward. So when image is taken with a pose far form this assumption, say, facing ground, lots of ground will not be classified correctly.

3.    In some cased when it is dark and ground and buildings merge together. It fail to detect ground. It happens rarely though. Most of time, it did amazingly well even when it is dark.

Some speed up for ground labeling software

1. Removed image poping part.

2. Removed vertical area sub-classification in ijcvTestimage.m.

Reduce the input resolution to 320x240 give as good result but much faster.

Due to the previous optimizations, processing time for a 320x240 image is 2.2 sec, which is a big boost from previous 15 secs.

APPgetLabeledimage2.m is modified so, besides a final labeling image output, it also output a raw ground probability map before any maxing between multiple labeling happening.

Shadow detection package

Download the package from this link: 

http://www.jflalonde.org/software/shadowDetection.zip

It depends on a external package, which it didn't mention at all in its README. The only clue is a few function call names. It turns out to be this graph cut package: 

http://vision.ucla.edu/~brian/gcmex-2.3.0.zip 

I am using version 1.9 though. You need to compile it in Matlab though with mex command:

compile_gc.m

(It seems that version 2.3 interface changed compare to 1.9 and you might need to do a bit hacking to make it work. Or you could find ver1.9. I don’t remember where do I find it. )

Add a line at the beginning of demoShadowDetection.m function: 

addpath('../GCmex1.9'); 

then: 

demoShadowDetection(dirName, imgName) 

It takes about 5 mins for image of size 800x600.

I modified the interface a little bit to be batch-run-friendly, but it should not be too different. You could all add a bit code in demoShadowDetection to save the result.

1.    In many cases, a blob of inconsistence color on the ground is not distinguishable from shadow even human, without high-level global reasoning.

2.    The algorithm seems prefer harsh change of luminance. When shadow edge blurs, there is higher probability that it is missed by the detector.

3.    Sometimes, obvious harsh edge goes undetected for no reason!

Lane detection package

cd src 

make release
