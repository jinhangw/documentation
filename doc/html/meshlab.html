Reconstruct a 3D image

1. open the file and import the right mesh file from VisualSFM result

Open Meshlab, file > open project > browse to the folder (name).cmvs > 00, double click to open the .rd.out file, then double click to open list.txt file. After a few seconds to allow the mesh load. Then delete the current mesh (by right click on the name from the right panel). Go to File > Import Mesh… > navigate to file Model > option-0000.ply.

2. Clean up the mesh. save it (either to the original mesh, or as a new mesh file by using file> export mesh as...).

3.0 Optional step. Go to Render > show vertex normal, and show arix. Go to Filters > Point set > Compute normals for point sets. set the viewpoint position to make sure the surface is properly recognized (when most normals vertex pointing upwards from the surface). Check the Flip normals w.r.t. viewpoint.

3. Filter > Point sets > Surface Reconstruction: Poisson. Set the parameters (12, 10, 2, 1). Apply.

3.1 The Poisson surfacing algorithm tends to produce bubble-like objects. You can remove the extraneous faces with “Filters > Selection > Select faces with edges longer than”. The

default value will usually remove most extraneous faces.


4. Delete the cloud point layer. Export the poisson mesh. Save the project as well.

5. Clean up the mesh using selection and delete. Save it. Then, go to Filters > Selection > Select non manifold Edges, and hit delete button. Save it.

6. Filters > texture > Parameterization + texturing from registered raster..  Set texture size as 4096 to get better results (4096 seems to come with more information, which is required by NEA workflow). Rename the texture name (optional). Check use distance weight, use image border weight, and use UV stretching (optional). Hit Apply. Save the layer as .obj. which can be opened in Blender.

7. Filter > Camera > Project active rasters color to current mesh, filling the texture. Rename the texture file, pixel size as 2048 or 4096 (optional). Check the boxes to use angle weight, distance weight,, image border weighter, and depth decontribute weight. Apply. Save the layer, as well as the project. Go to file > reload the layer. Switch from point view to Flat or Smooth view to see the color.

MeshLab crashes when..

no reason,

octree depth/solver parameters set too high for it to compute,

wrong workload command or work order,

forget to save after each step.


Meshlab unexpected (unespected) eof when opening a file:

possible reasons:

1. localization issue. If on ubuntu, try $LC_ALL=C meshlab to run the program

2. Try to open the file in another editor. Replace all dots with commas

3. The file is too big. In my case, I have trouble opening a 4.5 million vertices file. When I chopped it to 1.7 million, it loads fine.



How to get mesh for car scene in Meshlab (Inaccurate Normals result in absurd mesh!)

1. Load all option ply files. Then go to Filter > Flatten visible layers, check all four box. After applying this filter, the point cloud layers should be combined into one. Clean it up. Save it as option-all.ply. (If you see nothing after flattening the layers. Check if the rendering mode is ‘point’. Also, sometimes it has problem rendering the color, although the color information is not lost. Try Render > Color > per vertex.) Divide the scene. Keep the point cloud of each car in a separate ply file. The ground as a separate one.

2. Open the ply file in Matlab to calculate the center of the car (x0, y0, z0)

3. Open a car model. Render Axis.

4. Filter > Normals, Curvatures and Orientation > Compute normals for point sets. Set Neighbor num as 500 ( this value should be large enough to compute a reasonable uniformed normal for a facet). Check the box to ‘flip normals w.r.t viewpoint’. Select ‘View Pos’ from the drop down menu. For example, set (x0, 5000000, z0) to view from a far away point. Click OK. Chop off the points with wrong normals.

Tips: for the second and third view positions, set it as (x1, y1, z1) and (-x1, y1, -z1). It is very likely that these three pieces will cover the whole car.

5. Repeat step 4 for many times to make sure all points get a right normal (at least once). Import all these layers.

6. Filter > Mesh layer > Flatten visible layers. Check all four boxes. If the object is invisible after the process, make sure to switch back to ‘Points’ render on this new layer.

7. Filter > Remeshing, simplification and Reconstruction > Surface reconstruction: Poisson. I normally set the four values as 12, 10, 2, 1. (or 10, 6, 1, 1. Feel free to explore more)

More info on remeshing: http://meshlabstuff.blogspot.com/2010/07/remeshing-and-texturing-1.html

8. File > Export mesh as… Check boxes to save vertice flag/normal/color, face flag/normal…whichever is applicable. Uncheck the box of ‘binary’.


How to clean up point cloud in Meshlab

Use to select points on a plane. This is useful to remove points inside the car.

Press Shift to subtract selections (deselect). Press Ctrl to add selections (not very easy to control with plane selection mode, helpful in normal selection).

After selecting, click on the button again to stop selection mode. This is to be able to rotate the model and check, before removing the points.


Oct.2014 How to colorize lidar points -- workflow

1. Run VisualSFM on the images taken by Samsung camera. Combine all option.ply output files into option-all.ply. (see above, or manually edit them)

2. In CloudCompare, get the lidar data ready (chop it, save into ply ASCII). Open the lidar ply file and the option-all.ply. Align the lidardata.ply to option-all.ply.

3. Adjust the aligned lidardata.ply file to make sure 1) it has 6 columns (x,y,z as the first three column. the rest can be either r,g,b or nx,ny,nz). 2) it has no headers. The first line should be the total number of points. The point cloud data start from the second line. 3) Name it as lidar.ply.
For colorization code, please refer to the instructions here.

4. Copy the lidar.ply file into the colorization folder(/home/pork/Colorization/vis_sfm_copen). Remove any existing files. Copy the camera_2.txt file, text folder, and visualize folder from the VisualSFM result in step 1. They should locate in the dense reconstruction folder, for the same model which you used in step 2.

5. In a terminal, cd to the ~/colorization folder.
 ./taru to run the colorization code. Use the correct image width/height, focal length.

Note: focal length should be accurate, otherwise it stops at certain number, or it fails to even start to get the fov.

Note: No more than 200 images (1280x960) should be used. If too many images, it crashes when allocating memories. It reports segmentation fault.

6. Make a mesh out of the colorized aligned lidardata.ply point cloud. You can still clean up the points before meshing. Go to Filter > remeshing > reconstruction: Poisson.

7. Texture it.


How to prepare mesh for 3d printing

The file required by the 3d printer has to be stl format, 1000kb or less.

In Meshlab, use Quadric Edge Collapse Decimation to simplify a mesh. Refer to:

http://www.shapeways.com/tutorials/polygon_reduction_with_meshlab

From the menu, select Filters > Remeshing, simplification and construction > Quadratic Edge Collapse Detection. If your model is textured, there is also an option (with texture)

1.The target number of faces should be proportional to your desired file size. For example, a 190000-faces file has a size of 9000kb. So, the target number of faces should be around 20000, in order to get the size 9 times down. In most cases, a 20000-faces model will be around 1000bk.

2. Make sure to check the boxes for: Preserve Normal, Preserve Topology, Optimal position of qsimplified vertices, Planar simplification, Post-simplification cleaning

3. As usual, go to File > Export mesh as… Save it into .stl


How to create thick mesh plane for 3d printing

I had a problem that the ground is too thin to get a continuous 3d printing result. So, the idea is to get a thicker mesh surface.

http://meshlabstuff.blogspot.com/2009/04/creating-voronoi-sphere-2.html

1. Cut out the surface that you want to make thicker. This ensures that you have as less vertices as possible, so that Meshlab doesn’t crash.

2. Filter > Remeshing, Simplification and Reconstruction > Uniform Mesh Resampling. Set the offset as 0.5 (try out different values to get best result). Check the box for clean vertices, and absolute distance (you can explore other combinations. Multisample is recommended, yet it causes crash sometimes).

3. Use the Align function > manual rough glueing. Carefully align the offset layer to your desired position. Click on store transform.

4. Go to Filter > Mesh layers > Flatten visible layers. Save the combined mesh model.

5. You can prepare this model for 3D printing now.


Importing ASCII ply file into Meshlab

When importing, a window pomps out asking for:

number of lines in header: (16 in our case)

separation: (choose ; or . or space)

separation columns: x y z r g b nx ny nz reflectance (refer to the header, and find the right one that matches the parameter order in the header).


Notes:

Replace #QNAN with 0.

Replace all . with , if ERROR:unespect eof happens. It may be because of localization issues.

Check your data if part of the information failed to display (e.g. color of a certain part). The program omits invalid data and doesn’t always report error.

Align and Merge (with poisson filter) point clouds in Meshlab


Load the two meshes we wanted to align. Click on the Align button. Select the main mesh layer, and set as Glue Here Mesh. Select on the to-be-aligned layer, and click Point Based Glueing.

Select four points to align the two mesh. (check the box on the left buttom to enable scaling.)

Click Ok when we finished selecting the aligning points. Check the result. Redo the alignment until we get satisfying result.

Go to Filter > Mesh Layer > Flatten Visible Layers. Now we should have one combined mesh out of the aligned ones. Go to Filter > Remeshing, Simplification and Reconstruction > Surface Reconstruction: Poisson. Give the proper octree value and divide solver. Wait for it to process. Save it as AlignedMesh.ply.

Open a new Meshlab workplace, load the raster files of the original main mesh layer (Open project…). Then import AlignedMesh.ply(Import mesh…). Now delete the point cloud model.

Go to Filters > Textures > Parameterization, texturing from registered rasters. Select the textrue png file of BOTH main mesh and to-be-aligned mesh.

Save the work.


Note:

I had problem that the detailed one is denser than the main one!! The main one is not precise enough to conduct aligning. In this case, try to increase octree number to get denser mesh for main one. Practically,

1) Take pictures at proper intervals

2) Chop off the less important part of the main (large) point cloud data; so that there’s enough memory for a denser mesh.

Check the ‘keep unreferenced vertices’ box when flatten visible mesh layers, if all your faces are unreferenced.
