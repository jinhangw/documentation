Additional features for app

We want to extend the functionality of the app. For one thing, we want to make it as self-contained and automatic as possible to install it on the users vehicle with no or minimal user input needed. Another thing is that we like to get additional information like speed, windshield wiper on/off, headlights on/off and other things that measure the vehicle state or indicate driving conditions.

Vehicle ignition on/off

It is very important to know if the vehicle is being operated or not in order not to waste power and disk space. There are several options that might indicate if the vehicle is on or off:

  1. The power to the cigarette lighter: In some cars, the power to the cigarette lighter is switched off when the ignition is switched off. One would need to detect if the device is being charged or not. There certainly are indicators that show if the device is being charged.

  2. Vibrations: As soon as the engine is on there are vibrations of a few hertz. These might be detectable by the accelerometers.

  3. Ignition on/off indication from the OBDII: From other projects we know that the OBDII indicates ignition on/off

  4. GPS: If the vehicle is stationary, there will be no change in GPS. This probably unreliable, GPS is not always available and one could not distinguish between waiting at a red light and shutting off the engine. However, if the vehicles are always parked in the same area at the end of the day.



OBDII

From other projects we know that OBDII has a lot of information, e.g. ignition on/off and speed. We connected it to a data collection box via OBDII interface->serial port. There are serial-to-USB converters. But this seems to be a complicated way to do it. A more promising way might be to have a Wifi-OBDII or bluetooth-OBDII transeiver. Some are specifically made for smartphones (but several are only for iPhone, iPad)
