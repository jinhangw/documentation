=========Install VisualSFM, CMVS, and MeshLab==========

step 0: Check the system properties.

Go to Computer> system properties. In our case, the system type is 64-bit operating system.

step 1: Get VisualSFM windows binaries from here:
http://ccwu.me/vsfm/. Follow the instructions and save it to a folder of your choice.
step 2: Get the CMVS/PMVS binaries from here:

http://code.google.com/p/osm-bundler/downloads/detail?name=osm-bundler-pmvs2-cmvs-full-32-64.zip

And then put
cmvs.exe, pmvs.exe, genOption.exe and pthreadVc2.dll in the same folder
as VisualSFM.exe

You will find these files in the directories:
.../osm-bundler/osm-bundlerWin64/software/cmvs/bin    and
.../osm-bundler/osm-bundlerWin64/software/pmvs/bin

step 3: Download and install MeshLab on windows.

You might have errors (e.g. “missing Microsoft Visual C++ 2008 SP1 Redistributable Package (x64)”). Fix them and install.


=============VisualSFM============

Set Initial pairs in VisualSFM

In spanning tree view, right click two images to be the pair, then use SFM->More Functions->Set Initialisation Pair, then run the reconstruction again.


Visualsfm-How to use calibration parameters

1. from the menu bar, sfm > more functions > set calibration parameters (fx cx fy cy r)

2. sfm > more functions> use shared calibration

3. open the ini file, calculate and set the ratio.


How to calculate focal length, given image size

f1/f2==w1/w2 (to calculate the focal length of a different sized image, say f1 is fx)

cx=width/2

cy=height/2


Improve the 3D reconstruction

1. try "SfM->More Functions->Find More Points" and then "SfM->Resume SFM". Maybe a few more images can be registered.

2. In Spanning Tree view, press Up and Down to see what are the images registered in one model. This way, we may find the disconnect point and take more images.

Possible solutions not tried yet...

1. There is a hidden experimental feature you can try. Type "nvvirtual" in the status bar, and try resume your reconstruction.


2. Some tweaking with initialization can help a little bit, but I think it might be more effective to tweak the feature detection and matching. This would require you to reprocess from the beginning.

2.1. Detection. The default detection tries to limit the number of features. Try change the parameters to get more features:

    param_use_siftgpu 3

    param_use_siftgpu_customize -fo -1 -v 0 -maxd MAXD

    Here replace MAXD with something like twice of you image size, which would also require more GPU ram.

    Also read the issues about image sizes: http://www.cs.washington.edu/homes/ccwu/vsfm/doc.html#size

2.2. Matching.  The default is 8192 features. Try change the parameter to matching more features:

Change param_gpu_match_fmax to something larger. For example, if each image can have 16K features, you can specify 16000.



Potential problem with small GPU memory

By default, SiftGPU tries to fit your memory cap by down-sampling the images when needed. It is possible that you get much less features on small-memory GPUs, which produces different results on different machines.


You can check the remote/CPU solutions in FAQ, or


You can also try disable the down-sampling and re-run experiments.

  1, delete all the .sift and .mat files

  2, choose "Tools -> Enable GPU -> Customized Param"

     Add "-nomc" to the parameter list, which means "no memory cap"

  3, re-run matching and reconstruction.

  *, this does not guarantee to work because of the out-of-memory possibility.

  *, this may work, and may become extremely slow because of using virtual memory.

Image Requirement for Improving Reconstruction Result




Problems:

As shown above, images taken during different times of a day are not linked into one model. The reason might be the shadow?


1. at least two cameras. Best outcome is that the overlapped areas are reconstructed. It only works in very particular cases. Maybe, with same focal length, the angle difference (the directions of visible cameras) should be as small as possible but more than 10 degree. Two picture from very similar view point won’t work. Tow picture with angle more than 30 degree will not work in most cases.

2. More features would help to locate the camera. For example, if there is only one object (very less features), object should be more than 20 percent of the picture.

3. For cases with more than 2 images, one object has to show up in at least 3 images. i.e. overlap twice. Otherwise, it is very unlikely to get the images modeled into one model.

4. To use the least number of images, multi-perspective sets are better than parallel-perspective sets.

5. Reflections and shadows do affect the program’s judgement.

6. If camera move close to the objects in one single direction, for three pictures, the shot length of the farthest one should be long shot; and the three images taken in equal intervals. Three image combination works in very particular cases. Three pics all close (medium shot to close-up level) will not work. Three pictures all long shot (medium to long shot level) will not work. One of the pictures is three times farther than the other two, won’t work. As long as camera moves only in one direction, there seems to be a ‘blind area’ near the center. More pictures improve the reconstruction quality. For example, decrease the blind area.

7. The reconstruction is more noisy when camera directions are diverged, rather than pointing to a center. (example hall61_openCameraNoisy)

   Hall way scenario:

   Three difficulties for hallway reconstruction: 1) Since the images are taken inside the hall way, the camera directions are diverged, which gives more noise than camera pointing to the objects. 2) The hall way space is limited. It is impossible to move too far away. Thus more images are needed to meet the OVERLAP prerequisite. 3) The scene include a bright illuminating source.

   The best approach is to slice the hallway. For each slice, start by standing near side wall B, rotate the camera to take images of the side wall A( 6 images, about 15 degree interval); pan the camera to take the ceiling images until reaching side A; now stand near side A, rotate the camera and take 6-7 images of side B; step about 1 meter down side A to move to another ‘slice’, and repeat the process; after taking all slides, wrap the whole scene up by taking 12 images (with camera pointing 45 degree up) along the circumference of the scene. (example hall74)


8. (1280*1024 video frames. car scene in wean-nsh parking area)

Black-colored objects get worse reconstructed than light color ones (for both buildings and cars).

9 Correct camera parameters help improve the result. (1280*1024 video frames. car scene in wean-nsh parking area)

Before applying camera parameters,

i) the reconstruction is more noisy.

ii) there is noticeable mismatching (e.g. We put 3 triangle sign in the scene. We got 4 triangle reconstructed.)

iii) there is larger hole on the black car.

The above three points got improved after applying the camera parameters. Also, the image processing speeds up a little.

============================ CMPMVS ==============================


CMPMVS = Center for Machine Perception* Multi-View reconstruction Software

Install:

It only works on Window 7.

Read the readme.txt carefully. Make sure you have Visual Studio 2008 x64 redistributable and the correct version of DirectX installed.


Parameters:

meshEnergyOpt: doOptimizeOrSmoothMesh=TRUE will give you the result is very close to the highest possible accuracy (that can be obtained from your input images) anyway.
(The command line example: C:\user\...\cmpmvs\CMPMVS_6>CMPMVS.exe E:\...\00\mvs.ini DoGenerateVideo=True.)


How to use CMPMVS

In VisualSFM, from the menu SfM > Load NView Match, select your  NVM file from earlier project. Click Tab to see the dense 3D model after it is loaded. Click CMVS button, save as type NVM to CMPMVS. (This is to generate the input files for CMPMVS.)

In command line, cd  to the folder where CMPMVS executable is installed/located. To run the application, type CMPMVS.exe, followed by a space, followed by the directory of the mvs.ini file.

To find the directory of the mvs.ini file, browse to the .cmp folder > 00, copy the route and paste into the command line, and type the name of the mvs.ini file.

Hit Enter to run with default settings. You can also add more options to change parameters.

For example, put another single space, and paste the options you want. The options can be found in readme file under CMPMVS root folder.

Hit Enter to run CMPMVS with your settings.

The command line example: C:\user\...\cmpmvs\CMPMVS_6>CMPMVS.exe E:\...\00\mvs.ini DoGenerateVideo=True.



CMPMVS crash r128


Scenario 1: Absolute path issue


In the 00 folder look at the target mvs.ini file. Open it with wordpad or any text editing program. Note the directory:


[global]

dirName="C:\Users….\test.nvm.cmp\00\data\"

prefix=""

imgExt="jpg"

ncams=1475

width=1280

height=720

scale=1

workDirName="_tmp"

doPrepareData=TRUE

doPrematchSifts=TRUE

doPlaneSweepingSGM=TRUE

doFuse=TRUE

nTimesSimplify=10


and check whether it is the correct drive you typed in to the cmd or not. This might come from moving the folder (C:\ or K:\ or D:\, etc). Correct it and it will work the same.


Scenario 2: shared_calibration issue


It is safe to run VisualSFM steps with all default settings.

Based on our experience, when we check the option for using shared calibration in VisualSFM, CMPMVS crashes after about 40% of its processing.


CMPMVS Errors occur during solving maxflow


fix it by

1. Set:

doPrepareData=FALSE

doPrematchSifts=FALSE

doPlaneSweepingSGM=FALSE

doFuse=FALSE

in your ini file in order to skip some previous computation. Don't delete the _tmp directory.

2. Close all other applications and run it again. Don't open any other application during the computation.

3. Change planMaxPts from 3000000 to 2000000 and run it again. Don't open any other application during the computation.


Our biggest dataset so far is a set of 630 images, size 1280x960, with modified settings. And the images have not too many feature points.




Note:

If crash with modified settings, change:

doPrepareData=TRUE

doPrematchSifts=TRUE
