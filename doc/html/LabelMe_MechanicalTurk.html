====================Mechanical turk GUI============================

In general, we give qualifications. Approve all people for the qualification; aka give them the money. Then pick out people (80% for grade 1; 90% for grade 2) for the main task. Thus the bar of main task is set as grade equal or higher than 1.


Create and Manage projects from user interface

Under Create tab, we can create new project, or edit on existed projects.

For new projects:

Go to Other, so we can costumize our own template.

    Click on Advance after basic settings to remove Masters.

    Edit the html source. Insert an image (src = ‘image_url’). This should be corespond to the uploaded csv data file ( contains one column, first record is ‘image_url’, followed by all urls). If we need a second image, name it image_url2, and set another column in the csv data file. Mostly we put only one image in each HIT, to make things simple and clean.

    under Design layout tab, we can change the Frame height so people don’t have to scroll down to see the tasks.


Under Manage tab,

We can edit existed projects.

Publish batch:: when we can upload new data (a new csv file) for the existed template.

Go to Workers tab to assign qualification type (grade 1 or 2). We can also give Bonus ($0.10) or Block spammers.


To check the status of the tasks

We can cancel the task when it’s almost done (994/1000). (People will not be able to see the link any more on mturk.com)


Check and edit the result

Download the result as csv. (workid is important). In Excel, put x to Approved and upload csv; alternatively, we can click Approve all in the interface.

We can either edit the csv using excel or Python scripts from Srivatsan. (go into the labelme_mturk_python folder and use ../env/bin/python.Also, when you setup the task using the interface, like we did for the image categorization task, you have to use the interface to approve and finish the tasks - there are no scripts. You can also edit the results csv file manually and upload the results.)

Prepare the csv and task images


1. extract and select images

On the linux machine (server), create four folders under myfolder (/home/pothole/projects/snow_condition_detection/data/Images/). Each folder stores the extracted images from videos (four road conditions).
Run a script to remove images with very low brightness.

Manually throw out images in which the roads are not visible at all. (Keep the images where the roads are visible, though the road condition is difficult to tell.)

Select altogether 1000 images from the four folders equally. (This can be done by going into each folder, ctrl_scroll to minimize the images, and select by column)


2. transfer into the server folder

Click on the directory bar on Home, change it to /var/www/pothole.ahs.ri.cmu.edu/public_html/myfolder. Copy the 1000 images into this myfolder.

open a webpage, go to pothole.ahs.ri.cmu.edu/myfolder (and pothole.ahs.ri.cmu.edu/myfolder/xxxx.jpg)to test it. Also, test it on a different machine.


3. generate the csv

In Terminal, cd projects/Snow_Condition_Detection/code. Press Enter. git pull. Press Enter. Enter the password.

Open the create_main_task.py file (by typing subl . &) in a new terminal.

If not in snow_condition_detection/code - try public_html.

Edit the parameters:

(Note there is a / at the end)

In a new terminal, cd /var/www/pothole.ahs.ri.cmu.edu/public_html$

Now in folder publish_html, we should find myfolder, create_main_task.py, myfolder.csv, index.html, snow_condition_instructions.html. (Copy the create_main_task.py from the code folder. Edit on this one you copied.)



=======================Insert LabelMe into MTurk=========================

All source code of LabelMe is under projects/labelme_source. This should be kept intact. A copy of this folder on our server is under the directory: /var/www/pothole.ahs.ri.cmu.edu/public_html/labelme. We can modify codes in this folder to change labelme features loaded from our server.


All Mechanical Turk files are located at /home/pothole/aws-mturk-clt-1.3.1. The file mturk.properties is under /home/pothole/aws-mturk-clt-1.3.1/bin. It is a important file where access key, secret key are stored. It also contains code for switching between sandbox and mturk.


Use .sh to work with Mechanical Turk

example: pothole@pothole:~/aws-mturk-clt-1.3.1/bin$ ./getBalance.sh

Note, if under sandbox mode, the balance will be $10000.00.

You can use ApproveWork.sh to approve without removing. You have to give it an input file. If you go into the labelme_mturk_python directory, you will find a file name that is like task_name.success , where task_name is whatever name you used for the task. That file is the input to ApproveWork.sh script.



Create Instructions

The instructions are stored here:

/var/www/pothole.ahs.ri.cmu.edu/public_html/instructions


Pick the images

The images to be labeled are stored here: /var/www/pothole.ahs.ri.cmu.edu/public_html/labelme/Images. Create a folder to put all the images. Name the folder the same name as the task (e.g. newtask).


Create a yaml

Under home/projects/labelme_mturk_python, duplicate task_specific_conf.yaml. Rename it for the new task (newtask.yaml).

In the newtask.yaml, adjust the parameters:

name:

title_description:

(On MTurk, name and title description will be shown as a url on the left corner of the task. People can click )

assignment:

how many repeats (e.g. 20 repeats for each qualification image, 1 for main task image)

…

instructionlink:

This is the instruction page displayed before mturkers click ‘Accept HIT’. It starts with https.

(e.g. https://pothole.ahs.ri.cmu.edu/instructions/snow_night_main.html)

separate_annotations:

set 1 so people won’t see previous labels. Each worker get a folder created in labelme/annotation and labelme/images. 

set 0 when Assignment = 1.

Sandbox: 0/1

!!! Don’t forget to change /home/pothole/aws-mturk-clt-1.3.1/bin/mturk.properties

Generate HITs (officially on MTurk now!)

It’s important to do the following tasks in sequence:

1) go to /var/www/pothole.ahs.ri.cmu.edu/public_html/labelme/Annotations. Delete all the previous folders except the example folder. 

2) Label one image from sandbox main task. DO NOT submit. open the image in a new tab. view source. Ctrl+f, find ‘pothole.ahs…..’ which is the image link. Open the link in a new tab (to see a single image). Remove the mt attributes (to see the image with labelMe interface).

3) approve and remove everything from sandbox (qualification and main).

4) in both yaml, set sandbox_mode = 0.

5) comment out sandbox, recomment the mturk link in mturk.properties.

6) set up a new qualification ID from requester account (manage -> qualification type). Copy the ID into main.yaml

7) set up both qualification and main. Copy the links! Leave the two terminals open!

8) change the links in the instruction page.


Keep watching from requester mode, see if people submit works.

send emails on rejection. Don’t reject often. Peopel may sue you!:(


To restart server

cd projects/labelme_mturk_python/

type ../env/bin/python restart_redirecting_server.py newtask.yaml


After we get enough people

cd projects/labelme_mturk_python/

type ../env/bin/python approve_and_remove.py newtask.yaml

Remove the qualification as well as the main task. This approves their work without qualifying them. The labels get stored in the xml files. You can use the .input file generated by the scripts to look at the images and the labels for the main task
